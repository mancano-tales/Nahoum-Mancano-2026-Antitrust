Governança e Alinhamento Atômico
Humano-IA

O Agent Covenant é um framework de diretrizes, hooks e scripts projetados para alinhar agentes inteligentes (Cursor, Claude, Copilot) e manter a consistência de código e documentação.

Quando trabalhamos com múltiplos agentes de IA de forma paralela em um mesmo repositório, o contexto pode se degradar rapidamente. Mudanças destrutivas e commits sem rastreabilidade humana causam entropia no histórico. Este template introduz rituais de auditoria local de commits e autorregeneração estrutural de links de agentes.

Por que utilizar este modelo?

💡 Evita conflito de instruções: Seus agentes leem exatamente o mesmo arquivo de contexto ao mesmo tempo.
🛡️ Pre-commit rigoroso: Previne commits acidentais sem atualizações de changelogs ou planos de trabalho.
🔄 Autocorreção de links: Resolve quebras silenciosas provocadas por salvamento atômico de editores gráficos.

🧱 A Estrutura de Governança

O sistema é ancorado sobre três diretrizes operacionais fundamentais:

1. Fonte Única de Verdade (Contexto Canônico)

O arquivo CLAUDE.md na raiz do projeto é a fonte absoluta de instruções. Para garantir compatibilidade cruzada, o setup cria hard links físicos (AGENTS.md e .github/copilot-instructions.md) e uma junção NTFS (.agents/ apontando para .claude/). Isso garante que o Claude CLI, o Copilot e o Antigravity compartilhem a mesma memória em tempo real.

2. Roteiro de Planos YAML

Cada tarefa complexa exige a criação de um plano sob 9-vers/plan/ com cabeçalho YAML delimitando: 

---
tipo: Plano
titulo: "Correção da Barreira de Renda"
status: ATIVO
criado: 2026-07-11
concluido: null
agentes:
  orquestrador: "Claude Fable 5 (Claude Code)"
---

3. Commit Sincronizado (Co-committing)

Nenhuma linha de código ou texto entra no Git sem um registro associado no arquivo NEWS.md. O pre-commit hook bloqueia o commit caso arquivos de lógica tenham sido staged sem a devida atualização do changelog histórico.

⚡ Quick Start

Instanciar o template em seu ambiente de trabalho leva menos de um minuto:

  1. 1. Copie o template: Copie os arquivos da pasta do template para o diretório raiz do seu novo projeto.
  2. 2. Execute o script de Setup: Abra o terminal na raiz do projeto e execute o script correspondente ao seu sistema operacional:
    # No Windows (PowerShell como Administrador ou Modo Desenvolvedor):
    .\setup.ps1
    # No Linux / macOS (Terminal):
    chmod +x setup.sh
    ./setup.sh
  3. 3. Ative os Git Hooks: Copie os hooks da pasta hooks/ para a pasta oculta do Git:
    git config core.hooksPath hooks

🛠️ Ferramentas & Console

O ecossistema é mantido por dois utilitários programáticos nativos em R (Base R):

Validador de Consistência (validate-governance.R)

Audita e sincroniza o ecossistema. Ele possui dois modos:

Rscript tools/validate-governance.R: Valida o padrão de nomenclatura, checa conflitos de merge e garante que planos concluídos possuam logs salvos no inventário.

Rscript tools/validate-governance.R --sync: Reconstrói dinamicamente a tabela do índice plan/README.md, injetando as modificações cirurgicamente entre os delimitadores de tags.

Exportador de Conversas (export_conversa.R)

Cria logs autárquicos em Markdown a partir do histórico JSONL das IAs:

Rscript tools/export_conversa.R <session_uuid> [slug-do-capitulo]
